# Demo 资产清单（截图 / GIF / 一句话卖点）

目标：为 Upwork / Fiverr / Toptal / GitHub Sponsors 页面准备**可复制**的展示素材，让每次上架/投递的准备成本最低。

> 说明：本文件只描述「做什么 / 怎么做」，不包含任何账号信息。截图/GIF 等文件建议统一放到 `docs/assets/`。

## 1) 关键链接（直接复制）

- GitHub Repo： https://github.com/peng1233/offline-file-organizer
- 在线 Lite 版（GitHub Pages）： https://peng1233.github.io/offline-file-organizer/lite/
- BTC 打赏页： https://peng1233.github.io/btc-receive/

## 2) 30 字一句话卖点（选 1 条）

- 一键把杂乱文件自动归档到结构化目录，并生成可追溯的清单（快、干净、可复用）。
- 给普通用户的离线文件整理工具：打开 → 选择文件夹 → 一键归档 + 导出清单。

## 3) 截图清单（至少 3 张）

截图文件建议放在 `docs/assets/`：

1. `lite-home-v10.png`：Lite 首页（展示主要按钮/版本号）
2. `lite-after-run.png`：运行后结果（展示目标目录结构/导出提示）
3. `tc-donation.png`：打赏页（展示 BTC 地址/二维码）

### 截图注意

- 避免包含真实姓名、邮箱、路径里带个人信息、浏览器书签等
- 统一分辨率（例如 1280×720 或 1440×900）
- 示例文件/目录使用 `demo/`、`test/` 等非隐私命名

## 4) GIF / 视频（加分项）

目标：展示「选择文件夹 → 一键运行 → 输出归档结果 + 清单」。

- GIF：10–20 秒、可循环
- 视频：30–45 秒（Upwork/Fiverr 更友好，可附链接）

### Windows 录制建议

- Windows 11：`Win + G`（Xbox Game Bar）录屏
- 或使用 ScreenToGif（录屏后裁剪成 GIF）

## 5) 这些素材在页面上的放置位置

- Upwork Project Catalog / Proposal：1–3 张截图 + 1 个 GIF（或视频链接）
- Fiverr Gig：3–6 张图（可拼图）+ 可选短视频
- GitHub Sponsors：在 README / docs 中展示截图，并在 tiers 描述里引用
