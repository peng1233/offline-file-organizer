# Demo 资产清单（截图/GIF/一句话卖点）

目的：为 Upwork / Fiverr / Toptal / GitHub Sponsors 的页面准备 **可复用** 的展示素材，降低每次上架/投递的准备成本。

> 说明：本文件只给 怎么做/做什么，不包含任何账号信息；截图/GIF 等二进制文件建议放到 docs/assets/。

## 1) 必备链接（直接复制）

- GitHub Repo: https://github.com/peng1233/offline-file-organizer
- 在线 Lite 版（GitHub Pages）: https://peng1233.github.io/offline-file-organizer/lite/
- BTC 打赏页: https://peng1233.github.io/btc-receive/

## 2) 30 秒一句话卖点（任选其一）

- 把一堆乱文件按规则自动归档成清晰目录，并输出可追溯的清单（离线、本地跑、可复现）。
- 面向非技术用户的离线文件整理工具：一键打开、导入、生成归档结果 + 报告。

## 3) 截图清单（最小 3 张）

建议文件名（放 docs/assets/）：

1. lite-home.png：Lite 首页（包含入口按钮/版本号）
2. lite-after-run.png：运行后结果（展示输出目录结构/结果提示）
3. tc-donation.png：打赏页（展示复制地址/二维码区域）

### 截图注意

- 尽量裁掉浏览器地址栏的个人信息/书签栏。
- 统一分辨率（建议 1280×720 或 1440×900）。
- 若截图中包含示例文件名，避免隐私（用 demo/test 文件名）。

## 4) GIF/短视频（可选但加分）

目标：展示选择文件夹 → 一键运行 → 输出结果的闭环。

- GIF 10~20 秒，循环播放
- 或 30~45 秒短视频（Upwork/Fiverr 常用）

### Windows 录制建议

- Windows 11：Win + G（Xbox Game Bar）录制屏幕
- 或用免费的 ScreenToGif（体积小、导出 GIF 方便）

## 5) 交付物在页面上的使用位置

- Upwork Project Catalog / Proposal：放 1~3 张截图 + 1 个 GIF（或短视频链接）
- Fiverr Gig：主图 + 1 张流程图/步骤图（可用截图拼图）
- GitHub Sponsors：把截图放到 README 或 docs/，Sponsor tiers 里贴链接
