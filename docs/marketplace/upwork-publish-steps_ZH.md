# Upwork 发布步骤（Project Catalog 项目目录）

目标：用本仓库里已经准备好的可复制素材，在 Upwork 上创建一个**最小可用**的 Project Catalog 项目（建议先“保存为草稿”）。

本仓库所需素材：
- 链接集合（复制粘贴）：`docs/marketplace/links_EN.md`（英文为主，链接通用）
- 项目模板（正文/交付物/需求清单）：`docs/marketplace/upwork-project-catalog-template_EN.md`
- 短描述（字段受限时用）：`docs/marketplace/service-description-short_EN.md`
- 演示素材：`docs/assets/lite-demo.png`（或 `lite-demo.gif`）
- 可选封面图（生成的）：`docs/assets/fiverr-cover.png`

步骤：
1) 打开 Upwork -> Find Work -> Project Catalog（创建新 Project）。
2) 标题（Title）：
   - 从 `upwork-project-catalog-template_EN.md` 里选一个短、清晰的标题。
3) 分类 / 技能 / 标签：
   - 参考：`docs/marketplace/platform-keywords-tags.md`。
4) 项目描述（Project Description）：
   - 粘贴 `upwork-project-catalog-template_EN.md` 的完整描述。
   - 如果 Upwork 限制外链：只保留 **GitHub 仓库 + Demo 链接**。
5) 交付物 / 需求（Deliverables / Requirements）：
   - 直接复制模板里的 Requirements 清单。
   - 尽量简单：输入文件夹 + 命名规则 + 一个例子。
6) 图片（Images）：
   - 上传 `docs/assets/lite-demo.png`（推荐）。
   - 可选：上传 `docs/assets/fiverr-cover.png` 作为更干净的封面图。
7) 价格 / 周期（Price / Timeline）：
   - 参考：`docs/marketplace/pricing-delivery-tiers.md` 选择一个保守的基础档。
   - 不要承诺做不到的内容。
8) 预览检查：
   - 链接是否正确、图片是否显示、文案是否清晰。
   -（经验）如果平台对中文兼容不佳，优先使用英文模板避免乱码。
9) 保存为草稿（Save as DRAFT）：
   - 如果触发任何**付费/身份认证/绑定承诺**，不要点最终发布/提交。

备注：
- 一定要让 Demo + Repo 链接可见，方便买家自验证。
- 如果需要 100~200 词个人简介片段：用 `docs/marketplace/platform-profile-template.md`。
