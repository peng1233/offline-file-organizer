# GitHub Sponsors（开源赞助）开通清单（给维护者自己用）

> 目标：把“对外可发布资产 + 变现入口”做成可重复的闭环。
> 本文只写流程与可复制文案，不包含任何密钥。

## 1) 准备材料（10 分钟）

- 项目一句话定位（给陌生人看的）
- 3 个核心收益点（效率/省钱/省时间）
- 2~3 张截图（推荐：主页、典型操作、结果页）
- 赞助者能得到什么（哪怕是“优先支持/路线图投票/鸣谢”也行）

## 2) Sponsors 页面要写什么（模板）

### 标题（任选其一）

- Support Offline File Organizer
- Sponsor development of Offline File Organizer

### 简短说明（可直接粘贴）

Offline File Organizer is a small tool that helps you organize files locally (no cloud required). Sponsoring helps keep the project maintained, improves documentation, and funds new features.

### “赞助者回报”建议（不要承诺做不到的）

- Sponsors are listed in the README (optional)
- Priority on bug reports and questions
- Early access to planned features / roadmap updates

## 3) 推荐的赞助档位（示例，可按实际改）

- $3 / month — Coffee: keep the lights on
- $10 / month — Supporter: priority Q&A
- $25 / month — Pro: feature request discussion (no guarantees)

原则：档位越少越好；先跑通收款与沟通链路。

## 4) 仓库里要补的入口（最小改动优先）

- README 增加：
  - GitHub Sponsors 链接（如已开通）
  - BTC 打赏页：https://peng1233.github.io/btc-receive/
- docs/monetization/ 目录：
  - 放置“服务包/文案/交付清单/FAQ”

## 5) 上架后的验证（闭环证据）

- README 的链接点击可达
- Sponsors 页面文案与档位可见
- 有 1 个“外部用户路径”能走通：
  - 看到项目 → 理解价值 → 找到赞助入口 → 能发起赞助/打赏

## 6) 风险与注意

- 不要在任何文档里放 Token/私钥/敏感信息
- 不要承诺“定制功能一定做”，用“讨论/优先支持”措辞
- 变更尽量小：只增文档/入口链接，方便回滚
